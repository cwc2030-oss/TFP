

## Listing wizard debounced auto-save
- `app/dashboard/listings/[id]/edit/_components/use-autosave.tsx` — reusable `useAutosave({listingId, body, enabled})` hook + quiet `AutosaveIndicator`. Debounces 800ms, fires the SAME per-step `PATCH /api/listings/[id]` the manual Save uses. Skips initial mount (baseline = draft-loaded values), coalesces rapid edits, serializes overlapping saves, `flush()` wired to form `onBlur` for immediate save on leaving a field/clicking a link, and a `beforeunload` guard warns on hard refresh/close if a save hasn't landed. `markSaved()` lets manual Save update the baseline so the indicator doesn't flash "unsaved" before nav.
- Wired into `lease-terms-form.tsx` (step 2) and `content-contact-form.tsx` (step 3, `enabled={!isPublished}` — published mode is photos-only, no PATCH). Both forms now share a `buildBody()` used by manual save + autosave.
- Draft-only & safe: PATCH endpoint refuses non-DRAFT (409) and never accepts publish/lifecycle fields, so autosave can never auto-publish. Publish validation/required fields unchanged.
- The `new` flow step-1 property picker is the CREATE action (no draft exists yet), so autosave correctly does not apply there.
