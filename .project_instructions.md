### Terrain Intelligence Engine ‚Äî Ridge & Flow Quality (May 1, 2026)

#### Problem
Large territories (789ac, 756ac, 994ac, 2740ac) showed almost no terrain intel ‚Äî typically 1 spine, 1 secondary, 1 saddle regardless of acreage. Two root causes:
1. **Modal ridges endpoint returned 0 features** ‚Äî algorithm thresholds were tuned for mountain terrain, not Missouri's gently rolling hills (30m SRTMGL1 DEM).
2. **Synthetic backbone generator was hardcoded** to 1 primary + 1 secondary + 1 saddle, ignoring parcel size.
3. **Next.js routes sent overly restrictive params** (`min_length_m: 200`, `min_prominence_ft: 20`) to Modal, overriding even correct server-side defaults.
4. **No fallback when Modal returned 0 features** ‚Äî app accepted empty 200 OK without falling back to synthetic.

#### Fixes Applied (Session ~95)
- **`terrain_brain_v3_ridges.py` v1.1.0** (deployed to Modal):
  - Adaptive terrain classification: gentle (<100m range or <15m std) vs rugged
  - Gentle terrain: threshold lowered from p85 ‚Üí p65, weights shift toward elevation (0.35 vs 0.25)
  - `MIN_RIDGE_LENGTH_M`: 150m ‚Üí 60m
  - `MIN_PROMINENCE_FT_PRIMARY`: 20ft ‚Üí 8ft, secondary: 10ft ‚Üí 3ft
  - Fragment connection: 2x dilation for gentle terrain
  - Primary classification: score threshold p90 ‚Üí p80, length multiplier 1.5x ‚Üí 1.2x
  - Gap tolerance in `_order_points`: 200m ‚Üí 500m
  - Saddle detection: 5ft ‚Üí 3ft drop threshold
  - Extensive `[DIAG]` logging at every stage (TPI ranges, score distributions, cell counts)
- **`lib/ridge-extraction.ts`** ‚Äî Synthetic backbone now scales with acreage (multiple primaries, spurs, saddles proportional to territory size)
- **`app/api/ridge-spines/route.ts`** ‚Äî Fixed params sent to Modal (`min_length_m: 60`, `min_prominence_ft: 8`), added fallback when Modal returns 0 features
- **`app/api/terrain-flow/route.ts`** ‚Äî Same param fix for ridge data fetch within flow endpoint

#### Territory Save/Reload Bug Fix
