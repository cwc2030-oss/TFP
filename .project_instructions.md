## Listings (Lease Marketplace) — Chunk 1 shipped Apr 26, 2026
Landowner-side draft form. Full spec & manual test steps in `nextjs_space/docs/listings.md`.

### Schema (additive, applied via `prisma db push`)
- `model Listing` — anchored to `SavedProperty` via `savedPropertyId` FK (NOT a Report — no Report model exists in TFP). FK also to `User` via `ownerUserId`.
- Snapshot fields (all nullable): `state`, `county`, `acres`, `terrainScore`, `primaryMovement`, `bedAcres`, `funnelCount`. Pulled from SavedProperty.
- Drift tracker: `savedPropertyUpdatedAt` (compare against current SavedProperty.updatedAt).
- Lease fields: `askingPriceMin/Max`, `leaseType` (enum), `huntersMax`, `seasonAvailability` (string[]), `amenities` (Json).
- Content: `title`, `description` (Text), `photos` (string[]).
- Contact: `contactMethod` (enum), `contactEmail`, `contactPhone`.
- Status: `status` enum (default DRAFT), `publishedAt?`.
- Indexes: `(ownerUserId, status)` + `(state, status)`.
- Enums: `ListingStatus { DRAFT PENDING_REVIEW PUBLISHED LEASED EXPIRED WITHDRAWN }`, `LeaseType`, `ContactMethod`.

### OPSEC (HARD CONSTRAINT)
- Listing NEVER stores precise location. NO `centroidLat`, NO `centroidLng`, NO polygon, NO parcel ID column on Listing.
- `state` + `county` only — must match the public county-only map.
- 2 automated tests guard this (`__tests__/listing-opsec.test.ts`):
  1. Schema regex check on `prisma/schema.prisma` Listing block — fails if it sees `centroid|lat|lng`.
  2. Source check on `lib/listings.ts` `snapshotFromSavedProperty()` — same regex.
- If anyone adds lat/lng columns to Listing, the test suite breaks. DO NOT bypass these tests.

### API
- `GET /api/listings` — 401 if unauth. List owner's listings.
- `POST /api/listings` — 401 if unauth, 404 if not owner of SavedProperty. Body: `{ savedPropertyId }`. Creates DRAFT.
- `GET /api/listings/[id]` — 401/404. Returns listing + savedProperty include.
- `PATCH /api/listings/[id]` — 401/404. **409 if status !== DRAFT** (DRAFT-only updates in chunk 1).
- All routes use `Prisma.JsonNull` for amenities `null` translation (zod may produce `null`, Prisma JSON column requires `Prisma.JsonNull`).

### Pages
- `/listings` — server, lists owner's listings. 307 → `/login` if unauth.
- `/listings/new` — Step 1 (pick SavedProperty). Redirects to `/listings` if owner has 0 SavedProperties.
- `/listings/[id]/edit?step=1|2|3` — wizard resume.

### Tests

### Asset filter gotcha (env-level)
- The file-write layer auto-replaces `.jpg`/`.png` URL substrings inside `.ts`/`.tsx` strings with real CDN URLs (often produces unterminated strings).
- Workaround: write file via `python3 /tmp/script.py` heredoc → bypasses filter.
- For test fixtures with URLs, use `'https://cdn.example.com/listings/' + 'photo1' + '-asset'` (no image extension) to avoid filter triggering.

### Chunk 4 — Public Inquiry Flow (shipped Apr 26, 2026)

End-to-end hunter→landowner inquiry funnel on PUBLISHED listings. Hunter submits form → landowner gets email → hunter gets confirmation → owner sees in `/dashboard/inquiries`.

#### Schema
- `model Inquiry` — anchored to Listing via `listingId` FK, NO User FK on the hunter side (hunters submit publicly without accounts).
- Migration: `prisma/migrations/20260426154403_add_inquiry/migration.sql`.
- Fields: `id`, `listingId`(FK), `hunterEmail`(lowercased), `hunterName`, `hunterPhone?`, `preferredDates?`, `partySize`(default 1), `message`(Text), `status`(default NEW), `source?`, `utmSource/Medium/Campaign?`, `ipHash?`, `createdAt`, `repliedAt?`, `closedAt?`.
- Enum `InquiryStatus { NEW REPLIED CLOSED }`.
- Indexes: `[listingId, status]`, `[hunterEmail]`.
- Listing back-relation: `inquiries Inquiry[]` added.

#### OPSEC continuation
- Inquiry stores hunter contact + message ONLY. No location columns.
- Email bodies use `state/county/acres/lease/price` ONLY via `EmailListingContext` interface. The interface enforces this at the type level — emails CANNOT receive lat/lng/coords.
- Unit tests guard: `__tests__/inquiry-api.test.ts` "OPSEC continuation" group inspects rendered email HTML, asserts no `centroidLat|centroidLng|latitude|longitude|parcel|address|geom|polygon|coordinate` substrings, no lat/lng decimal patterns.
- `inquiryInputSchema` is `.strict()` — rejects unknown fields like `centroidLat`, `lat`, `lng`, `coords`, `parcel`, `address`, `geom`.

#### Helpers
- `lib/email.ts` — Abacus `sendNotificationEmail` wrapper. Default `abacusTransport`; `setEmailTransport(stub)` / `resetEmailTransport()` for test injection. `sendEmail({to, subject, html, bcc?, replyTo?})` — BCC is a 2nd API call (Abacus accepts one recipient per call). `replyTo` not on the wire (Abacus doesn't support it); landowner email body shows hunter email as a `mailto:` link.
- `lib/inquiry.ts` — `inquiryInputSchema` (strict zod), `ipHash()`, `resolveClientIp(headers)` (leftmost x-forwarded-for → x-real-ip), `landownerSubject() / buildLandownerEmail()`, `hunterSubject() / buildHunterConfirmationEmail()`. Constants `IDEMPOTENCY_WINDOW_MS=7d`, `RATE_LIMIT_MAX=3`, `RATE_LIMIT_WINDOW_MS=1h`. `esc()` HTML-escapes via `replace(/.../g, ...)` (NOT `replaceAll` — TS target=es2020).

#### API
- `POST /api/listings/[id]/inquire` — PUBLIC, no auth. Order: parse → validate → honeypot silent drop → listing PUBLISHED-only 404 → rate-limit 429 → idempotency 7d → create → fire emails (errors logged, never 500). Returns `{ok:true, inquiryId, slug, idempotent?}`. GET returns 405.
- `POST /api/dashboard/inquiries/[id]/status` — owner-only. Body `{status: "REPLIED"|"CLOSED"}`. 401/404(collapsed: not-found OR not-yours)/400/409(already-target)/200. Stamps `repliedAt`/`closedAt`.

#### Pages
- `/listings/[slug]/inquire` — REPLACED chunk-3 placeholder. Server-validates PUBLISHED, renders header + `<InquiryForm />`. 404 for any non-PUBLISHED.
- `app/listings/[slug]/inquire/_components/inquiry-form.tsx` — client form. Posts to `/api/listings/{listingId}/inquire`. Honeypot field `website` (off-screen). Pulls UTMs from URL. Fires `trackEvent('listing_inquiry', ...)`. 429/404 surface as user-friendly errors.
- `/dashboard/inquiries` — auth-gated, redirects to `/login?callbackUrl=%2Fdashboard%2Finquiries`. Table: Date | Listing | Hunter | Status | Actions. Newest first. Scoped to owned listings.
- `_components/inquiry-row-actions.tsx` — NEW→[Mark Replied, Close], REPLIED→[Close], CLOSED→none. POSTs to status route, calls `router.refresh()` on success.
- `_components/inquiry-message-disclosure.tsx` — 140-char preview + Show more/less.
- Dashboard "Inquiries" link added to `/dashboard` between "My Listings" and "New Report".

#### Email contract
- **Landowner inbox fallback chain:** `Listing.contactEmail` → `User.email` → `console.warn` and skip (inquiry still saved, dashboard still shows it; no crash).
- BCC: `INQUIRY_BCC=clark@terrafirma.partners` env var. ALL landowner emails BCC clark for ops visibility.
- Hunter confirmation copy: subject `"We forwarded your inquiry — {county}, {state}"`, "what happens next" body with 24-72hr reply window, no listing-detail link back.

#### Status transitions
- `NEW → REPLIED` (stamps `repliedAt = now`).
- `NEW → CLOSED` (stamps `closedAt = now`).
- `REPLIED → CLOSED` (stamps `closedAt = now`).
- 409 if already in target.
- Returning to `NEW` is NOT exposed via this endpoint by design.

#### Idempotency / rate-limit
- Idempotency: same `(listingId, hunterEmail)` within 7d returns existing inquiryId, `idempotent: true`, NO email re-fire.
- Rate limit: max 3 per hour per `ipHash`. 429 + `Retry-After` header. Salted via `INQUIRY_IP_SALT` (sha256). Raw IPs NEVER stored.

#### Env vars added
- `INQUIRY_BCC=clark@terrafirma.partners`
- `INQUIRY_IP_SALT=<sha256 hex>` (set once via `openssl rand -hex 32`).

#### Tests (chunk 4) — 40 new

- `__tests__/inquiry-api.test.ts` (23 tests) — mocks `next-auth`, injects `setEmailTransport(captureTransport())`. Covers status guard (DRAFT/PENDING_REVIEW/LEASED/WITHDRAWN → 404 no leak, 0 emails, 0 rows), idempotency 7d (same inquiryId, no re-fire), rate-limit 3/hr (429 + Retry-After, leftmost xff IP, ipHash stored not raw), email firing (3 calls: landowner-To + landowner-BCC + hunter), owner-inbox fallback (contactEmail → owner.email → warn+skip), OPSEC keys/decimals, honeypot silent 200, status route 401/404/200/409/400.
