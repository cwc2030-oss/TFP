

## Auto parcel-shape listing image (OPSEC-safe)
- `lib/parcel-listing-image.ts` generates a public listing hero image from the anchored SavedProperty's `parcels` JSON (Feature-wrapped GeoJSON — extractor unwraps Feature/FeatureCollection).
- OPSEC: renders parcel OUTLINE only on a neutral dark-stone branded bg (emerald `#10b981` stroke). NO satellite/basemap/roads/labels, NO coordinates, NO GPS/EXIF metadata. Polygon is normalized (equirect projection → centroid-center → PCA principal-axis rotation → long-axis horizontal → deterministic de-mirror → scale-to-fit) so the outline is NOT a georeferenced tracing reverse-matchable to public parcel DBs. County/state text is allowed (county-level is OPSEC-safe); precise location is never used.
- Output: 1600×1000 PNG via `next/og` `ImageResponse` (no native deps). Uploaded to public S3 key `listings/{id}/parcel-shape.png` with `?v=` cache-bust.
- Hooked into: `app/api/listings/route.ts` (POST create — attaches as photo #1 so zero-photo owners pass the review-publish gate, which is a CLIENT gate disabled at photos.length===0) and `app/api/listings/[id]/publish/route.ts` (safety net + regenerate with county/state now set). `ensureAutoListingImage` is idempotent + non-fatal; never overwrites owner-uploaded photos (detects our auto image via `isAutoParcelImage` = url includes `/parcel-shape.png`).
- Fallback when geometry missing: data-only "Terrain Certified" card (grade/acres/region) instead of blocking publish.
- Note: `next/og` only resolves inside Next's bundler — test via a dev-server route, not standalone node.
