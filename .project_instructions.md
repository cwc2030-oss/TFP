## v4-fix11: Camera-Synced Stand Markers
- **Two-phase scaling**: During camera motion → CSS `transform: scale()` on outer `el` (GPU-accelerated, no SVG rebuild). On camera settle → rebuild SVGs at final zoom for crispness.
- **rAF throttling**: Only one `requestAnimationFrame` in flight at a time during `move` events
- **Entry transition cleanup**: CSS transitions from staggered fade-in are removed 50ms after completion to prevent interference with zoom scaling
- **SVG rebuild threshold**: Only rebuilds if zoom delta > 0.3 from last rebuild (avoids unnecessary work on small pans)
- **`will-change: transform`** added during motion, removed on settle
- **Direction indicators** (tapered pole) are part of SVG, inherently synced with marker
- **Diagnostics**: `[STAND-DIAG] CAMERA MOVING`, `[STAND-DIAG] MARKER SYNC UPDATE`, `[STAND-DIAG] CAMERA SETTLED`, `[STAND-DIAG] MARKER SETTLED`
- **Refs**: `lastSVGRebuildZoom` tracks zoom at which SVGs were last built; `el.dataset.refScale` stores per-marker reference scale

## v4-fix12a: Normalized Stand Marker Zoom Behavior
- **Problem**: Stands 3-7 drifted off-parcel faster than stands 1-2 during zoom because of variable hitbox sizes (80/72/62px) causing different CSS transform behavior.
- **Fix 1 — Uniform hitbox**: All markers now use `HITBOX_SIZE = 80` regardless of tier. Visual SVG still varies (top=60, secondary=52, tertiary=42).
- **Fix 2 — Disc-aligned transform-origin**: `transform-origin` computed per marker so CSS `scale()` pivots on disc center (cy=40 in viewBox), not element center. Stored in `el.dataset.discPctY`.
- **Fix 3 — Marker offset**: Mapbox `Marker({ anchor: 'center', offset: [0, -yOffset] })` shifts marker so disc center (not element center) sits at the geo-coordinate.
- **Fix 4 — Rebuild normalization**: Phase 2 rebuild (on camera settle) recomputes `transform-origin` and calls `marker.setOffset()` (cast as `any` — method exists at runtime but not in TS types).
- **Constants**: `HITBOX_SIZE = 80`, `DISC_CENTER_PCT = 40`, `el.dataset.baseSize` stores per-marker visual base size.
