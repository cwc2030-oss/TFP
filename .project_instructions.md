### Saddle Quality Filter — Fewer, Defensible Saddles (2026-05-02)
- **Root cause**: `filterSpinesByQuality()` only filtered RIDGES — saddle_nodes passed through completely unfiltered from Modal AND synthetic pipeline
- **NEW FUNCTION**: `filterSaddlesByQuality()` in `lib/ridge-extraction.ts` — 5-stage pipeline:
  1. **Prominence filter**: ridgeDropFt ≥ 20ft (SADDLE_MIN_PROMINENCE_FT)
  2. **Ridge proximity filter**: must be within 150m of a ridge spine (SADDLE_MAX_DIST_FROM_RIDGE_M)
  3. **Confidence filter**: ≥ 0.3 (SADDLE_MIN_CONFIDENCE) — for Modal-provided confidence scores
  4. **Spacing filter**: greedy-pick best saddles first, enforce 300m minimum spacing (SADDLE_MIN_SPACING_M)
  5. **Density cap**: max 1.5 saddles per km of total ridge length (SADDLE_MAX_PER_KM_RIDGE)
- Sort score for greedy pick: 60% prominence + 30% proximity + 10% confidence
- Applied in BOTH `fetchRidgeSpines` client path AND `/api/ridge-spines` server route
- Returns `SaddleDebugPayload` exported type with full per-candidate diagnostics
- `[SaddleDebug]` console emission in intel/page.tsx: colored header, full JSON, console.table of all candidates
- Server-side `terrain_debug.pipeline_steps.saddle_filter` shows raw → final counts + all stage counts
- Helper: `pointToSegmentDistM()` — perpendicular distance from point to line segment for ridge proximity
